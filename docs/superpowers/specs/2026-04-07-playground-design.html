Playground 模型对比测试平台设计 ​

概述 ​

一个面向开发者设计的模型对比测试平台，支持同时对多个 AI 模型发起独立 Prompt 测试，快速对比输出结果。

核心功能 ​

  • Prompt 对比：输入同一个 Prompt，并行对比多个模型的输出
  • 模型参数调试：配置 Think 开关、温度、Top_p 等参数
  • 混合模型支持：同时支持 Ollama 本地模型和 API 模型（OpenAI、Anthropic 等）

用户场景 ​

  • 开发者调试：快速对比不同模型对同一 Prompt 的输出质量
  • 单轮独立 Prompt，不支持多轮对话链

组件说明 ​

1. 头部导航栏 ​

  • Logo + 产品名称
  • Ollama 连接状态指示（绿色/红色）
  • API 服务状态指示

2. Prompt 输入区 ​

  • 多行文本输入框
  • Monospace 字体
  • 支持长文本输入

3. 模型选择器 ​

  • Checkbox 分组显示
  • Ollama 模型和 API 模型分组
  • 支持多选

4. 参数面板 ​

参数类型默认值
Think 开关Checkbox开启
温度 (Temperature)Slider/Input0.7
Top_pSlider/Input0.9
系统提示词Textarea可选

5. 运行按钮 ​

  • 主按钮样式
  • 点击后触发所有选中模型并行执行
  • 运行中显示 loading 状态

6. 输出卡片 ​

每个模型对应一张卡片，显示：

  • 模型名称 + 来源标签
  • 输出内容（Markdown 渲染）
  • Token 消耗数量
  • 响应耗时

7. 状态指示器 ​

  • Ollama：未连接/已连接
  • API：未配置/已配置

数据流 ​

用户输入 Prompt
       ↓
用户选择模型 + 配置参数
       ↓
点击"运行" → 并行请求多个模型
       ↓
各自返回输出 → 更新对应输出卡片
       ↓
展示结果（输出 + 性能数据）

错误处理 ​

错误场景处理方式
Ollama 未启动提示"请先启动 Ollama"，禁用 Ollama 模型
API Key 无效显示认证错误，不阻塞其他模型
单模型请求失败对应卡片显示错误信息，其他模型正常显示

技术实现要点 ​

  • 前端框架：React + TypeScript
  • 状态管理：Zustand（轻量级）
  • Ollama 通信：REST API (localhost:11434)
  • API 模型：统一接口适配不同 provider
  • 并行请求：Promise.allSettled（单个失败不阻塞其他）

设计原则 ​

  • 效率优先：开发者追求快速对比，界面简洁直接
  • 信息完整：输出内容 + 性能数据同时展示
  • 容错友好：单模型失败不影响整体测试流程
