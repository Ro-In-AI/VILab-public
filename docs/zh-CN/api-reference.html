API 参考 ​

GET /health ​

返回服务元信息和当前公开暴露的 STT 模型别名。

鉴权：

  • 无需鉴权

返回示例：

json
{
  "serviceId": "uuid",
  "version": "0.2.4",
  "publicModel": "vilab-local-stt"
}

publicModel 始终反映当前的 STT 路由结果：

  • Cloud 模式下，它对应当前云端 STT provider/model 的公开别名
  • server-local 模式下，它对应服务端本地 worker 的公开别名，例如 vilab-local-stt

GET /docs/ ​

返回 VILab 文档站点。桌面客户端和无头服务端复用同一份 VitePress 文档构建产物；无头 Docker 镜像默认从 /usr/local/share/vilab/docs 挂载到 /docs/。

鉴权：

  • 无需鉴权

示例：

bash
curl http://127.0.0.1:9876/docs/

GET /v1/models ​

返回 VILab 当前对外暴露的公开模型列表。

鉴权：

  • Authorization: Bearer <external_api_key>

返回示例：

json
{
  "object": "list",
  "data": [
    {
      "id": "vilab-local-stt",
      "object": "model",
      "created": 0,
      "owned_by": "vilab"
    }
  ]
}

如果当前 STT mode 是 server-local，data 会返回服务端公开的本地模型别名，例如 vilab-local-stt。模型是否真的可用，需要通过 /admin/models 或 vilab doctor 检查 worker 状态。

POST /v1/audio/transcriptions ​

接收 multipart form data，并返回转录结果。

鉴权：

  • Authorization: Bearer <external_api_key>

表单字段：

  • file：必填，音频文件。当前这一期只接受 WAV。
  • model：可选，默认使用 /health 返回的当前公开 STT 模型别名
  • language：可选
  • response_format：可选，默认 json，也支持 text

当 STT mode 为 server-local 且 provider 为 sensevoice 时，这个接口会由 vilab-server 鉴权并接收音频，再转发给 SenseVoice worker。

JSON 返回：

json
{
  "text": "transcribed content"
}

纯文本返回：

  • 设置 response_format=text
  • 返回类型会变成 text/plain; charset=utf-8

错误码：

  • 400：缺少文件或模型不受支持
  • 401：Bearer token 缺失或无效
  • 502：当前 STT 运行时转写失败

/admin/* ​

管理员接口只用于宿主管理。

示例：

  • GET /admin/status
  • GET /admin/providers
  • PUT /admin/providers/{id}
  • GET /admin/routing
  • PUT /admin/routing
  • GET /admin/models
  • POST /admin/models/{id}/download
  • POST /admin/models/{id}/download/start
  • GET /admin/models/{id}/download/status
  • POST /admin/models/{id}/activate
  • GET /admin/workers
  • POST /admin/workers/sensevoice/reload
  • GET /admin/api-keys
  • POST /admin/api-keys
  • GET /admin/initial-api-key
  • POST /admin/api-keys/{id}/revoke

鉴权：

  • Authorization: Bearer <adminKey>

GET /admin/initial-api-key 只返回首次启动自动生成并写入 initial-api-key.json 的那把初始 external API key。如果该文件已被删除，会返回 404；这时应使用 POST /admin/api-keys 创建新 key。

GET /admin/workers ​

返回服务端已知 worker 的健康状态。当前首个 worker 是 sensevoice。

鉴权：

  • Authorization: Bearer <adminKey>

返回示例：

json
[
  {
    "id": "sensevoice",
    "kind": "sensevoice",
    "url": "http://vilab-sensevoice:9877",
    "reachable": true,
    "status": "ok",
    "model": "sensevoice-small",
    "modelInstalled": false,
    "error": null
  }
]

GET /admin/models ​

返回服务端模型状态。当前用于查看 SenseVoice 模型目录是否存在，以及该模型是否可被 worker 激活。

鉴权：

  • Authorization: Bearer <adminKey>

返回示例：

json
[
  {
    "id": "sensevoice-small",
    "name": "SenseVoice Small",
    "providerId": "sensevoice",
    "workerId": "sensevoice",
    "installed": false,
    "active": false,
    "modelDir": "/models/sensevoice-small",
    "status": "not_installed",
    "error": null
  }
]

POST /admin/models/{id}/download 和 POST /admin/models/{id}/activate 会转发到对应 worker。当前 worker v1 支持下载 sensevoice-small，会把 model.int8.onnx 和 tokens.txt 保存到模型目录。默认下载源是 HuggingFace；如果服务器网络受限，可以通过 VILAB_SENSEVOICE_MODEL_BASE_URL 改成内部镜像。

vilab init 和 vilab models download 会优先使用异步下载接口：先调用 POST /admin/models/{id}/download/start 创建任务，再轮询 GET /admin/models/{id}/download/status 显示进度条。状态响应会包含 bytesDownloaded、totalBytes 和 currentFile。

下载成功示例：

json
{
  "ok": true,
  "status": "installed",
  "model": "sensevoice-small",
  "modelDir": "/models/sensevoice-small",
  "bytes": 239549735
}
