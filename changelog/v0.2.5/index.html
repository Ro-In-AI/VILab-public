VILab 0.2.5

VILab 0.2.5

本次更新

  • 完成 Remote-first + Embedded fallback 架构收口：新用户默认连接远程 VILab Server，旧桌面用户升级时保留本机内嵌服务模式。
  • 统一 Public API 契约，vilab-server 与桌面内嵌服务共享 /health、/v1/models、/v1/audio/transcriptions 的鉴权、错误响应、模型列表和转写响应行为。
  • 收紧 external API key 与 Admin Key 边界：普通客户端、SDK 和业务集成只使用 external API key；Admin Key 只用于 /admin/*。
  • 增强 TypeScript/Python SDK：模型 owner 字段稳定映射，转写支持 JSON/text 返回，401 错误信息更可读。
  • 补齐 Docker server smoke，对 server-local SenseVoice worker 转写路径做发布前验证。
  • 合入 Windows IME 阶段性成果，包括 Windows 输入法工程、候选窗、诊断和发布准备脚本。

重点验证

  • Remote 模式缺少 external API key 时不会发起无效连接测试，并给出本地提示。
  • /v1/* 只接受 external API key，拒绝把 Admin Key 当作客户端 token 使用。
  • Embedded 与 Server 的 Public API contract tests 保持一致。
  • Docker server smoke 覆盖 vilab-server、vilab CLI、vilab-sensevoice-worker 和 mock transcription。
  • macOS/Windows 桌面能力继续保留在 Tauri 宿主与内嵌服务，不迁移到无头 Server。

已知说明

  • Remote 模式在本版本接管 STT；rewrite、prompt、dictionary/hotword 服务端化会在后续阶段继续推进。
  • 本机内嵌服务仍用于单机离线、LAN、Prompt Lab、输入法桥接和本地模型 fallback。
